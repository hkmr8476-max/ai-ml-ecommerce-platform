# ML AI E-Commerce Platform

Production-oriented monorepo for an AI-powered e-commerce platform with a Next.js frontend, FastAPI backend, and Python ML engine.

## Architecture Overview

- **Frontend (`/frontend`)**: Next.js 14 + TypeScript + TailwindCSS + Redux Toolkit.
- **Backend (`/backend`)**: FastAPI REST API (JWT auth, catalog/orders/recommendation/search/payment endpoints).
- **ML Engine (`/ml-engine`)**: Recommendation, demand forecasting, fraud detection, dynamic pricing, and search ranking modules.
- **Data Layer (`/database`)**: PostgreSQL schema and Alembic migration bootstrap.
- **Caching/Queue (`redis`)**: Session/rate limit cache and async event support.
- **Search (`/search-engine`)**: Elasticsearch integration contracts.
- **Storage**: S3-compatible object storage integration points.
- **DevOps (`/docker`, `.github/workflows`)**: Containerized local stack and CI pipeline.

## Key Features

### E-Commerce
- Authentication (JWT + OAuth placeholder)
- Product catalog, categories, filters, variants
- Cart + checkout + payments (Stripe/PayPal stubs)
- Orders and order history
- Reviews & ratings
- Admin dashboard API endpoints

### AI/ML
- Collaborative, content-based, and hybrid recommendations
- Semantic-like query scoring for product search
- Demand forecasting (time-series baseline)
- Fraud risk scoring for suspicious orders
- Dynamic pricing recommendations
- User segmentation for personalized marketing

## Quickstart

### 1) Prerequisites
- Docker + Docker Compose
- Python 3.11+
- Node 20+

### 2) Local environment
```bash
cp .env.example .env
```

### 3) Run full stack
```bash
docker compose -f docker/docker-compose.yml up --build
```

### 4) Access services
- Frontend: `http://localhost:3000`
- Backend API: `http://localhost:8000`
- Swagger: `http://localhost:8000/docs`

## Environment Variables
See `.env.example` for complete configuration (database, JWT, OAuth, Stripe, PayPal, Redis, Elasticsearch, S3).

## API
- OpenAPI auto-generated by FastAPI.
- Versioned routes under `/api/v1/*`.

## Security
- Password hashing using bcrypt.
- JWT access tokens.
- SQLAlchemy parameterized queries.
- CORS, basic rate limiting middleware, and secure settings placeholders.

## Testing
- Python: `pytest tests/backend tests/ml`
- Frontend: `npm test` (Jest scaffold)

## Deployment
- **AWS**: ECS/EKS + RDS + ElastiCache + OpenSearch
- **Vercel**: frontend deployment with backend URL env
- **Docker server**: compose-based deployment
- **Kubernetes (optional)**: manifests scaffold in `scripts/deploy_k8s.sh`
